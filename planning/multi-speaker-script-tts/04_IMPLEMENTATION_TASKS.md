# 다화자 대본 TTS 구현 작업 목록

## 1. 구현 목표

이 문서는 실제 코딩 단계에서 바로 사용할 수 있는 작업 목록입니다.
1차 구현은 `MVP 완성 + 기본 검증`까지를 목표로 합니다.
단, 구조는 장기적으로 줄 단위 재생성, 미리듣기, 버전 관리, 프로젝트 대시보드까지 확장 가능해야 합니다.

## 2. 작업 단위 요약

### 2-1. 데이터 처리

- 대본 파서 작성
- 화자 목록 추출 로직 작성
- 화자 설정 검증 로직 작성
- 작업 폴더 및 manifest 저장 로직 작성
- 프로젝트형 메타데이터 구조 설계
- 줄별 고유 ID 체계 설계

### 2-2. TTS 생성

- 저장된 voice prompt 기반 줄별 생성 함수 작성
- 화자별 후처리 옵션 적용
- 줄별 결과 저장
- 전체 합본 생성

### 2-3. UI

- 새 탭 추가
- 대본 입력 UI 추가
- 화자 추출 버튼 추가
- 동적 화자 설정 UI 추가
- 상태/로그/결과 표시 UI 추가
- 향후 줄 단위 대시보드 확장이 가능한 UI 구조 설계

### 2-4. 검증

- 정상 케이스 테스트
- 형식 오류 테스트
- 누락 파일 테스트
- 다중 화자 생성 결과 점검

## 4. 세부 작업 순서

### 단계 1. 파서와 검증 로직 먼저 구현

- `화자: 대사` 파서 작성
- 줄 번호 포함 구조체 정의
- 줄 고유 ID 정의
- 화자 목록 추출
- 오류 메시지 포맷 정의

완료 기준:
- 문자열 입력만으로 파싱 결과를 얻을 수 있음

### 단계 2. 줄별 음성 생성 함수 구현

- 저장된 voice prompt 파일 로드
- 한 줄 텍스트 생성
- 줄별 wav 파일 저장
- 생성 결과 메타데이터 반환

완료 기준:
- 단일 화자/단일 줄 테스트 통과

### 단계 3. 다줄 순차 실행 구현

- 파싱 결과를 순서대로 처리
- 진행 로그 수집
- 실패 정책 적용
- 작업 폴더 생성
- 줄별 메타데이터 누적 저장

완료 기준:
- 여러 줄 대본으로 연속 생성 가능

### 단계 4. 합본 생성 구현

- 줄별 wav를 순서대로 읽기
- 무음 구간 삽입
- 최종 `final_mix.wav` 저장

완료 기준:
- 합본 파일 재생 가능

### 단계 5. manifest 저장 구현

- 작업 메타데이터 JSON 저장
- 줄별 생성 성공/실패 정보 기록
- 향후 버전 관리 확장을 고려한 구조로 저장

완료 기준:
- 작업 폴더만 봐도 생성 내역 파악 가능

### 단계 6. UI 연결

- 새 탭 추가
- 입력 컴포넌트 연결
- 화자 추출 결과를 UI에 반영
- 생성 이벤트와 백엔드 함수 연결
- 결과 리스트를 향후 줄 단위 카드/표로 확장 가능하게 설계

완료 기준:
- 브라우저에서 전체 기능 실행 가능

### 단계 7. 오류 처리와 사용자 메시지 정리

- 파싱 오류 메시지
- 누락된 화자 설정 메시지
- 없는 파일 경고
- 모델 미로드 상태 경고

완료 기준:
- 실패 원인을 사용자가 바로 이해 가능

## 5. 테스트 체크리스트

### 5-1. 정상 케이스

- 2명 화자 대본 생성
- 3명 화자 대본 생성
- 줄별 파일 저장 확인
- 합본 파일 저장 확인

### 5-2. 오류 케이스

- 빈 대본 입력
- `:` 없는 줄 포함
- 대사 없는 줄 포함
- 화자 매핑 누락
- 존재하지 않는 `.pt` 파일 지정

### 5-3. 경계 케이스

- 화자가 한 명뿐인 경우
- 매우 긴 대본
- 같은 화자가 연속 10줄 이상 등장
- 빈 줄이 중간중간 섞인 대본

## 6. 우선순위

우선순위 `상`
- 파서
- 화자 매핑 검증
- 줄별 생성
- 합본 저장
- 프로젝트 메타데이터 구조

우선순위 `중`
- 화자별 후처리 옵션
- manifest 저장
- 로그 가독성 개선
- 줄별 결과 구조화

우선순위 `하`
- 특정 줄 재생성
- 프로젝트 저장/불러오기
- CSV/JSON 확장 포맷
- 복사 버튼
- 버전 히스토리 UI

## 7. 구현 전에 먼저 결정할 항목

아래는 실제 코딩 시작 전에 확정하면 좋은 결정 사항입니다.

1. 대본 형식은 1차에서 `화자: 대사`만 허용할지
2. 화자 없는 줄은 오류로 막을지
3. 실패 시 전체 중단이 기본인지
4. 줄별 파일과 합본 파일을 둘 다 기본 저장할지
5. 화자별 후처리 옵션을 1차부터 넣을지

추천 기본값:

- 대본 형식: `화자: 대사`
- 화자 없는 줄: 오류
- 실패 정책: 즉시 중단
- 출력: 줄별 + 합본 둘 다 저장
- 후처리 옵션: 1차에 포함

## 8. 완료 정의

아래 조건을 만족하면 1차 구현 완료로 볼 수 있습니다.

- 새 탭에서 다화자 대본 입력 가능
- 화자 추출 가능
- 화자별 voice prompt 지정 가능
- 대본 전체를 순서대로 생성 가능
- 줄별 wav 저장 가능
- 최종 합본 wav 저장 가능
- 오류 메시지가 사용 가능 수준으로 정리됨

그리고 아래 구조적 조건도 만족하는 것이 좋습니다.

- 줄별 고유 ID가 존재함
- 프로젝트 메타데이터 파일이 남음
- 나중에 줄 단위 재생성 기능을 붙일 수 있는 저장 구조를 사용함
