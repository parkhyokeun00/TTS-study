# 다화자 대본 TTS UI 화면 초안

## 1. UI 추가 위치

기존 [tts_app.py](/f:/abc/qwen-tts/tts_app.py)에는 현재 다음 탭이 있습니다.

- `실시간 음성 복제`
- `음성 프롬프트 저장/재사용`
- `음성 대 음성 변조 (RVC 실험)`

새 기능은 같은 앱 안에 새 탭으로 추가하는 것이 자연스럽습니다.

추천 탭 이름:
- `다화자 대본 생성`

## 2. 화면 목표

사용자가 아래 과정을 한 화면 안에서 끝낼 수 있어야 합니다.

1. 대본 붙여넣기
2. 화자 자동 감지
3. 각 화자에 음성 팩 연결
4. 생성 옵션 지정
5. 실행
6. 결과 다운로드

장기적으로는 아래 작업도 같은 흐름 안에서 지원할 수 있어야 합니다.

1. 줄별 미리듣기
2. 마음에 안 드는 줄만 다시 생성
3. 대사 텍스트 복사
4. 음성 파일 경로 복사
5. 줄별 버전 비교
6. 프로젝트 저장 후 나중에 다시 이어서 작업

## 3. MVP 화면 구성

### 3-1. 상단 설명 영역

- 짧은 안내 문구
- 대본 형식 예시 표시

예시 문구:

```text
각 줄을 "화자: 대사" 형식으로 입력하세요.
예) 나레이터: 오늘은 새로운 기능을 소개합니다.
```

### 3-2. 좌측 입력 영역

필수 입력 UI:

- `대본 입력` 멀티라인 텍스트박스
- `화자 추출` 버튼
- `출력 작업 이름` 텍스트박스
- `줄별 파일 저장` 체크박스
- `전체 합본 저장` 체크박스
- `줄 사이 무음(ms)` 슬라이더
- `생성 시작` 버튼

### 3-3. 우측 화자 매핑 영역

화자 추출 후 동적으로 생성되는 영역입니다.

화자별 행 구성 예시:

- 화자 이름 표시
- `voice prompt` 파일 선택
- 속도 슬라이더
- 피치 슬라이더
- 끝음 처리 드롭다운
- 끝음 길이 슬라이더

예:

```text
[나레이터]
- voice prompt: [파일 선택]
- 속도: 1.0
- 피치: 0.0
- 끝음 처리: natural
- 끝음 길이: 180
```

### 3-4. 하단 결과 영역

- 처리 상태 텍스트박스
- 줄별 진행 로그
- 생성된 줄별 파일 목록
- 최종 합본 오디오 플레이어
- 최종 합본 파일 다운로드

## 4. 장기 확장 UI 방향

초기에는 단순 탭으로 시작하되, 최종적으로는 `생성기 + 편집기 + 대시보드`가 결합된 형태를 목표로 합니다.

### 4-1. 줄 단위 결과 대시보드

줄별 카드 또는 표 형태로 아래 기능을 가질 수 있습니다.

- 줄 번호
- 화자
- 대사
- 현재 선택 버전
- 오디오 미리듣기
- `다시 생성`
- `대사 복사`
- `파일 경로 복사`
- `파일 열기`
- `합본 포함/제외` 토글

### 4-2. 버전 관리 영역

한 줄을 여러 번 생성할 수 있으므로 버전 개념을 둘 수 있습니다.

예:
- `line_012_v1`
- `line_012_v2`
- `line_012_v3`

사용자는 그중 하나를 현재 채택 버전으로 선택합니다.

### 4-3. 프로젝트 사이드바

장기적으로는 좌측 또는 우측에 프로젝트 개요 패널을 둘 수 있습니다.

- 프로젝트 이름
- 총 줄 수
- 완료 줄 수
- 실패 줄 수
- 마지막 렌더 시각
- 최종 합본 상태

## 5. 사용자 흐름

### 4-1. 기본 사용 흐름

1. 모델 로드
2. 새 탭 진입
3. 대본 붙여넣기
4. `화자 추출` 클릭
5. 감지된 화자별로 음성 팩 선택
6. 생성 옵션 설정
7. `생성 시작` 클릭
8. 줄별 로그 확인
9. 결과 다운로드

### 4-2. UI 상호작용 포인트

- `화자 추출`을 누르면 화자 매핑 폼이 자동 생성
- 대본을 바꾼 뒤 다시 추출하면 매핑 영역 갱신
- 기존에 매핑한 화자 이름이 같으면 가능하면 설정 유지

## 6. 레이아웃 초안

```text
------------------------------------------------------------
[다화자 대본 생성]

설명:
각 줄을 "화자: 대사" 형식으로 입력하세요.

------------------------------------------------------------
| 대본 입력                          | 화자별 음성 팩 설정 |
|-----------------------------------|---------------------|
| [멀티라인 텍스트박스]             | [나레이터]          |
|                                   | voice prompt 선택   |
| [화자 추출]                       | 속도 / 피치 / 끝음  |
|                                   |                     |
| 작업 이름                         | [여자1]             |
| 줄별 파일 저장                    | voice prompt 선택   |
| 전체 합본 저장                    | 속도 / 피치 / 끝음  |
| 줄 사이 무음(ms)                  |                     |
| [생성 시작]                       | [남자1]             |
|                                   | voice prompt 선택   |
------------------------------------------------------------
| 상태 / 로그                                             |
| [진행 로그 텍스트박스]                                  |
------------------------------------------------------------
| 결과                                                    |
| [최종 합본 오디오 플레이어]                              |
| [줄별 파일 목록]                                        |
------------------------------------------------------------
```

| 줄별 결과 대시보드                                                   |
|---------------------------------------------------------------------|
| 001 | 나레이터 | 대사 미리보기 | 재생 | 다시 생성 | 복사 | 채택버전 |
| 002 | 여자1    | 대사 미리보기 | 재생 | 다시 생성 | 복사 | 채택버전 |

| 최종 합본                                                           |
| [합본 오디오 플레이어] [합본 다시 만들기] [내보내기]                |

## 7. MVP에서 단순화할 부분

- 화자 수 제한: 우선 제한 없음으로 두되 실제 테스트는 3~6명 중심
- 줄별 개별 미리듣기: 초기 버전에서는 생략 가능
- 화자 색상 태그: 초기 버전에서는 생략 가능
- 드래그 앤 드롭 재정렬: 생략
- 줄별 버전 관리: 초기 버전에서는 생략 가능
- 복사/열기 유틸 버튼: 초기 버전에서는 생략 가능

## 8. 확장 UI 아이디어

### 7-1. 프로젝트형 UI

향후에는 단순 탭이 아니라 작은 프로젝트 편집기처럼 확장할 수 있습니다.

- 대본 저장
- 화자 사전 저장
- 이전 작업 재불러오기
- 특정 줄만 재생성
- 줄별 미리듣기
- 줄별 복사 버튼
- 채택 버전 선택

### 8-1. 편의 기능 대시보드

실사용 편의성 향상을 위해 아래 버튼들을 줄 단위로 제공할 수 있습니다.

- `재생`
- `다시 생성`
- `대사 복사`
- `파일 경로 복사`
- `폴더 열기`
- `합본에서 제외`
- `이 버전 채택`

### 7-2. 테이블형 대본 편집기

장기적으로는 텍스트박스 대신 표 형태가 더 편할 수 있습니다.

열 예시:
- 순서
- 화자
- 대사
- 감정
- 속도
- 상태

하지만 1차 버전에서는 자유 텍스트 입력이 구현 부담이 적습니다.

### 8-2. 로컬 전용 작업 흐름

이 프로그램은 로컬 전용 사용을 전제로 하므로 아래 UX가 잘 맞습니다.

- 로컬 파일 경로 직접 사용
- 작업 폴더 바로 열기
- Windows 탐색기에서 결과 확인
- 캐시 파일 재사용
- GPU 메모리 상황에 따라 생성 정책 조절

## 9. 사용자 안내 문구 제안

### 8-1. 도움말

```text
대본은 한 줄에 한 대사씩 입력하세요.
형식은 "화자: 대사" 입니다.
예)
나레이터: 오늘은 새로운 기능을 소개합니다.
여자1: 첫 장면을 시작하겠습니다.
```

### 8-2. 오류 메시지 예시

- `3번째 줄에 ':' 구분자가 없습니다.`
- `화자 '여자1'에 연결된 voice prompt가 없습니다.`
- `모델을 먼저 로드해주세요.`
- `최종 합본 생성 중 오디오 길이 정보가 올바르지 않습니다.`
