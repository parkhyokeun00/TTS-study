# 실시간 음성 복제 사용 설명서

이 문서는 `tts_app.py`의 첫 번째 탭인 `실시간 음성 복제` 기능만 처음부터 끝까지 설명합니다.

이 기능으로 할 수 있는 일:
- 내가 입력한 문장을 원하는 목소리 느낌으로 읽게 만들기
- 짧은 참조 음성을 기준으로 비슷한 화자 음색 만들기
- 결과 음성을 `.wav` 파일로 저장해 다시 듣거나 편집 프로그램에 넣기

## 1. 이 기능이 정확히 무엇인가요?

`실시간 음성 복제`는 다음 순서로 동작합니다.

1. 사용자가 읽힐 문장을 입력합니다.
2. 사용자가 기준이 될 `참조 음성`을 넣습니다.
3. 프로그램이 참조 음성의 화자 특성을 분석합니다.
4. 입력 문장을 그 화자 느낌으로 새로 읽은 음성을 생성합니다.

쉽게 말하면:
- `원고`는 새로 읽을 내용
- `참조 음성`은 따라갈 목소리 기준

## 2. 먼저 준비해야 할 것

처음 사용자에게는 아래 4가지를 먼저 안내하는 것이 좋습니다.

### 2-1. 필수 준비물

- Windows PC
- Python 설치 환경
- 인터넷 연결
- 참조용 음성 파일 1개 또는 마이크

### 2-2. 권장 참조 음성 조건

좋은 결과를 원하면 참조 음성은 아래 조건에 가까울수록 좋습니다.

- 길이: 5초에서 20초 정도
- 사람 목소리가 또렷하게 들릴 것
- 배경음악, 효과음, 박수 소리, 여러 사람 대화가 적을 것
- 너무 작은 소리나 찢어진 음질이 아닐 것
- 가능하면 실제로 말한 문장을 알고 있을 것

## 3. 설치 방법

이 기능은 TTS 전용 패키지를 먼저 설치해야 합니다.

```powershell
pip install -r requirements-tts.txt
```

권장 사항:
- 가능하면 TTS 전용 가상환경을 따로 사용하세요.
- GPU가 있으면 더 빠릅니다.
- 처음 실행 시 모델 다운로드 때문에 시간이 걸릴 수 있습니다.

## 4. 실행 방법

프로그램 실행 명령:

```powershell
python tts_app.py
```

현재 이 저장소에서 별도 가상환경 실행 예시:

```powershell
.\.venv-rvc\Scripts\python.exe tts_app.py
```

실행되면 브라우저가 자동으로 열리며, 기본적으로 `127.0.0.1:7862`를 사용합니다.

포트가 이미 사용 중이면:
- `7863`
- `7864`
- 그 다음 빈 포트

순서로 자동 변경됩니다.

## 5. 화면에서 먼저 해야 할 일

프로그램을 열면 탭으로 들어가기 전에 위쪽 공통 영역부터 설정해야 합니다.

### 5-1. 모델 선택

초보자에게 가장 권장하는 선택:

- `Qwen3-TTS-1.7B Base (음성 복제)`

이유:
- 이 프로젝트에서 음성 복제용으로 가장 자연스럽게 연결되어 있습니다.
- 참조 음성 기반 복제와 프롬프트 저장 기능이 모두 이 흐름에 맞춰져 있습니다.

### 5-2. 모델 로드

`모델 로드` 버튼을 누르세요.

상태창에 아래와 비슷한 문구가 나오면 정상입니다.

- 로드 완료
- 사용 디바이스가 `GPU (CUDA)` 또는 `CPU`

주의:
- 첫 로드는 매우 느릴 수 있습니다.
- CPU만 사용하는 경우 기다리는 시간이 길어질 수 있습니다.

### 5-3. 출력 언어

한국어 콘텐츠를 만들 것이라면:

- `출력 언어`를 `한국어`로 둡니다.

다른 언어도 선택할 수 있지만, 초보자 배포용 기본 설명은 한국어 기준으로 안내하는 것이 안전합니다.

## 6. 탭 1 `실시간 음성 복제` 사용 순서

아래 순서대로 따라 하면 됩니다.

### 6-1. `합성할 원고` 입력

이 칸에는 새로 읽힐 문장을 넣습니다.

예시:

```text
안녕하세요. 오늘은 처음 사용하는 분도 쉽게 따라 할 수 있는 음성 복제 방법을 안내해 드리겠습니다.
```

처음에는 너무 길게 쓰지 말고 1문단 정도로 시작하는 것을 권장합니다.

### 6-2. `참조 음성` 넣기

방법은 2가지입니다.

- `upload`: 기존 음성 파일 업로드
- `microphone`: 지금 직접 녹음

초보자에게는 보통 파일 업로드가 더 안정적입니다.

지원 흐름:
- WAV 파일
- 마이크 녹음

실제 내부에서는 스테레오가 들어와도 자동으로 한 채널로 정리됩니다.

### 6-3. `참조 음성 텍스트` 입력

이 칸은 매우 중요합니다.

의미:
- 참조 음성 안에서 실제로 말한 문장을 적는 칸입니다.

예를 들어 참조 음성이 아래 문장을 읽은 파일이라면:

```text
안녕하세요. 반갑습니다. 오늘도 좋은 하루 보내세요.
```

가능하면 같은 내용으로 정확하게 적는 것이 좋습니다.

효과:
- 발음, 억양, 화자 특성을 더 안정적으로 따라갑니다.
- 일반적으로 `x-vector only`보다 결과가 더 좋습니다.

### 6-4. `x-vector only 모드 사용` 선택 여부

이 체크박스는 두 가지 방식 중 하나를 고르는 옵션입니다.

체크하지 않는 경우:
- 정밀 복제 모드
- `참조 음성 텍스트`가 필요함
- 더 권장되는 방식

체크하는 경우:
- 참조 음성만으로 화자 특성을 추정
- 참조 텍스트 없이 사용 가능
- 편하지만 품질과 일관성이 떨어질 수 있음

초보자 권장:
- 가능하면 체크하지 말고 `참조 음성 텍스트`를 함께 넣으세요.

### 6-5. `음성 생성` 클릭

버튼을 누르면 프로그램이 다음 작업을 수행합니다.

1. 원고를 너무 길면 여러 구간으로 나눕니다.
2. 참조 음성을 분석합니다.
3. 각 구간을 새 음성으로 생성합니다.
4. 여러 구간이면 이어 붙여 하나의 파일로 만듭니다.
5. 결과를 `outputs/` 폴더에 `.wav`로 저장합니다.

정상 완료 시 결과창에는 대체로 아래와 같은 뜻의 메시지가 표시됩니다.

- 원본 음성 생성 완료
- 어떤 모드로 생성했는지
- 몇 개 구간으로 나누어 만들었는지

## 7. 생성 결과 확인 방법

`생성된 원본 음성` 영역에서 바로 재생할 수 있습니다.

동시에 파일도 저장됩니다.

저장 위치:

- [outputs](/f:/abc/qwen-tts/outputs)

파일 이름 특징:
- 입력 문장의 앞부분을 바탕으로 자동 생성
- 뒤에 고유 식별 문자열이 붙음
- 확장자는 `.wav`

예시 형태:

```text
안녕하세요_오늘은_ab12cd34.wav
```

## 8. 후처리 기능 사용법

원본 음성이 만들어진 뒤 바로 아래 `원본 생성 후 후처리` 영역을 사용할 수 있습니다.

이 기능은 목소리 자체를 새로 복제하는 것이 아니라, 이미 생성된 음성 파일을 다듬는 기능입니다.

### 8-1. `속도`

- 기본값: `1.0`
- 값이 커질수록 더 빠르게 재생됩니다.
- 값이 작을수록 천천히 들립니다.

처음 추천:
- `0.95`에서 `1.05` 사이

### 8-2. `높낮이`

- 반음 단위 조절
- 양수: 더 높게
- 음수: 더 낮게

처음 추천:
- `-1.0`에서 `+1.0` 사이에서만 소폭 조절

### 8-3. `끝음 처리`

선택 가능한 대표 의미:

- `기본`: 손대지 않음
- `부드럽게 마침`: 끝부분을 부드럽게 줄임
- `빠르게 감쇠`: 끝을 빨리 정리
- `여운 추가`: 끝을 살짝 늘림
- `자연스럽게 마침`: 조금 더 자연스러운 종료 느낌

### 8-4. `끝음 길이(ms)`

- 끝부분에 얼마나 길게 효과를 줄지 정하는 값입니다.
- 숫자가 클수록 끝음 처리 영향이 더 길어질 수 있습니다.

### 8-5. `후처리 미리듣기`

이 버튼을 누르면:

1. 원본 음성을 불러옵니다.
2. 위 옵션을 적용합니다.
3. 새 미리듣기 파일을 다시 저장합니다.

출력:
- `후처리 적용 음성`
- `후처리 결과`

저장 파일도 `outputs/` 폴더에 `.wav`로 남습니다.

## 9. 후처리 프리셋 저장 방법

같은 설정을 반복 사용할 예정이라면 프리셋을 저장하세요.

순서:

1. `속도`, `높낮이`, `끝음 처리`, `끝음 길이`를 원하는 값으로 맞춥니다.
2. `프리셋 이름`에 알아보기 쉬운 이름을 적습니다.
3. `현재값 저장`을 누릅니다.

예시 이름:
- 유튜브 나레이션
- 차분한 안내 음성
- 밝은 홍보 톤

프리셋은 다음 파일에 저장됩니다.

- [outputs/postprocess_presets.json](/f:/abc/qwen-tts/outputs/postprocess_presets.json)

나중에 같은 탭에서:

1. `저장된 프리셋` 선택
2. `프리셋 불러오기`

순서로 다시 적용할 수 있습니다.

## 10. 초보자용 가장 쉬운 추천 사용 흐름

처음 배포 시 아래 흐름을 그대로 안내하면 실패가 적습니다.

1. 프로그램 실행
2. `Qwen3-TTS-1.7B Base (음성 복제)` 선택
3. `모델 로드`
4. `출력 언어`를 `한국어`로 확인
5. `합성할 원고`에 1문단 입력
6. 잡음 적은 참조 음성 업로드
7. 그 음성의 실제 발화 문장을 `참조 음성 텍스트`에 입력
8. `x-vector only`는 끔
9. `음성 생성`
10. 결과를 듣고 필요하면 후처리

## 11. 잘 되는 사용 예

좋은 예:
- 또렷한 1인 음성
- 조용한 방에서 녹음한 파일
- 참조 음성과 참조 텍스트가 정확히 일치
- 원고를 1문단씩 나누어 생성

좋지 않은 예:
- 배경음악이 큰 파일
- 두 명 이상이 겹쳐 말하는 파일
- 참조 텍스트가 실제 음성과 다름
- 한 번에 너무 긴 원고를 넣음

## 12. 자주 막히는 문제와 해결 방법

### 12-1. `먼저 TTS 모델을 로드해주세요!`

원인:
- 위쪽에서 `모델 로드`를 누르지 않은 상태

해결:
- 모델 선택 후 `모델 로드` 먼저 실행

### 12-2. `참조 음성을 업로드하거나 녹음해주세요.`

원인:
- 참조 음성이 비어 있음

해결:
- 파일 업로드 또는 마이크 녹음 후 다시 시도

### 12-3. `정밀 voice clone 모드에서는 참조 음성 텍스트가 필요합니다.`

원인:
- `x-vector only`를 끄고도 참조 텍스트를 입력하지 않음

해결:
- 참조 텍스트를 적거나, 정말 필요하면 `x-vector only`를 켬

### 12-4. 모델 로드가 너무 느립니다

가능한 원인:
- 첫 다운로드
- CPU 사용 중
- 네트워크 속도 문제

확인:
- 상태창에서 `GPU (CUDA)` 또는 `CPU` 표시 확인

### 12-5. 마이크가 동작하지 않습니다

확인:
- 브라우저 마이크 권한 허용
- 주소가 `localhost` 또는 `127.0.0.1`인지 확인

## 13. 결과 파일은 어디에 모이나요?

이 기능으로 만들어지는 대표 파일:

- 원본 음성: `outputs/*.wav`
- 후처리 음성: `outputs/*.wav`
- 후처리 프리셋: `outputs/postprocess_presets.json`

배포 시 사용자에게 꼭 안내할 점:
- 작업 결과는 브라우저 안에만 있는 것이 아니라 실제 파일로 저장됩니다.
- 음성 파일은 나중에 영상 편집 프로그램이나 오디오 편집 프로그램에서 그대로 사용할 수 있습니다.

## 14. 한 줄 요약

이 탭은 `참조 음성 + 새 원고`를 넣고 바로 음성을 만드는 기능입니다. 초보자에게는 `참조 음성 텍스트를 함께 넣는 일반 모드`를 기본 사용법으로 안내하는 것이 가장 안전합니다.
