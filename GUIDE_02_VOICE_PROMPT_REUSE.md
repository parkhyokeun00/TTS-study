# 음성 프롬프트 저장 및 재사용 사용 설명서

이 문서는 `tts_app.py`의 두 번째 탭인 `음성 프롬프트 저장/재사용` 기능을 초보자 기준으로 자세히 설명합니다.

이 기능으로 할 수 있는 일:
- 한 번 분석한 목소리를 저장해 두기
- 다음부터는 참조 음성을 다시 넣지 않고 같은 화자 느낌으로 새 문장 읽기
- 같은 목소리로 여러 콘텐츠를 반복 제작하기

## 1. 이 기능이 왜 중요한가요?

`실시간 음성 복제`는 매번 참조 음성을 다시 넣어야 합니다.

반면 `음성 프롬프트 저장/재사용`은:

1. 참조 음성을 한 번 분석하고
2. 그 결과를 `.pt` 파일로 저장한 뒤
3. 나중에 그 파일만 다시 불러와 새 원고를 읽게 합니다

즉, 자주 쓰는 화자를 `재사용 가능한 음성 설정 파일`로 보관하는 기능입니다.

배포용 관점에서 이 기능은 특히 아래 사용자에게 유용합니다.

- 같은 나레이터 목소리로 여러 영상을 만드는 사용자
- 매번 파일 업로드를 반복하기 싫은 사용자
- 화자별 음성 자산을 정리해서 관리하려는 사용자

## 2. 먼저 알아둘 핵심 개념

이 탭에는 크게 두 단계가 있습니다.

### 2-1. 1단계: 음성 프롬프트 저장

입력:
- 참조 음성
- 참조 텍스트

결과:
- `voice prompt` 파일 생성
- 확장자: `.pt`

### 2-2. 2단계: 저장한 프롬프트로 새 음성 만들기

입력:
- 저장된 `.pt` 파일
- 새 원고

결과:
- 같은 화자 느낌으로 새 `.wav` 생성

이 기능도 TTS 기능이므로 동일한 설치가 필요합니다.

```powershell
pip install -r requirements-tts.txt
```

## 5. 1단계: 음성 프롬프트 저장 방법

탭 이름:

- `음성 프롬프트 저장/재사용`

위쪽 첫 영역에서 먼저 아래 항목을 입력합니다.

### 5-1. `프롬프트용 참조 음성`

이 칸에는 목소리 기준이 될 음성을 넣습니다.

방법:
- 파일 업로드
- 마이크 녹음

권장:
- 1명이 또렷하게 말한 음성
- 잡음이 적은 파일
- 너무 짧지 않은 구간

### 5-2. `프롬프트용 참조 텍스트`

이 칸에는 참조 음성 속 실제 문장을 적습니다.

예:

```text
안녕하세요. 제품 사용 안내를 시작하겠습니다. 아래 순서대로 천천히 따라 해 주세요.
```

좋은 사용법:
- 실제 음성과 문장을 최대한 맞추기
- 띄어쓰기와 문장부호까지 너무 크게 다르지 않게 적기

### 5-3. `x-vector only 모드 사용`

이 옵션은 참조 텍스트 없이도 저장할지 여부를 정합니다.

체크 해제:
- 일반 권장 모드
- 참조 텍스트 필요
- 품질이 더 안정적일 가능성이 큼

체크:
- 참조 텍스트 없이 저장 가능
- 더 간단하지만 품질과 재현성이 떨어질 수 있음

초보자에게는 기본적으로 체크 해제를 권장합니다.

### 5-4. `음성 프롬프트 저장`

버튼을 누르면 프로그램이:

1. 참조 음성을 분석합니다.
2. 재사용 가능한 내부 프롬프트 데이터를 만듭니다.
3. 그 데이터를 `.pt` 파일로 저장합니다.

정상 완료 시:
- `프롬프트 상태`에 성공 메시지 표시
- `저장된 voice prompt` 영역에 저장된 파일이 연결됨

## 6. 저장된 파일은 어디에 생기나요?

이 기능이 저장하는 핵심 파일은 `.pt` 파일입니다.

저장 위치:

- [outputs](/f:/abc/qwen-tts/outputs)

파일 이름 형태:

```text
voice_prompt_xxxxxxxx.pt
```

의미:
- 이 파일 안에는 목소리 재사용에 필요한 프롬프트 정보가 들어 있습니다.
- 원본 음성 파일 자체를 그대로 복사해 넣는 개념과는 다릅니다.

## 7. 초보자용 가장 쉬운 저장 절차

1. 프로그램 실행
2. 모델 선택 후 `모델 로드`
3. `출력 언어`를 `한국어`로 확인
4. `음성 프롬프트 저장/재사용` 탭 이동
5. `프롬프트용 참조 음성` 업로드
6. `프롬프트용 참조 텍스트` 입력
7. `x-vector only`는 끔
8. `음성 프롬프트 저장` 클릭
9. 저장 성공 메시지 확인
10. 생성된 `.pt` 파일 위치를 확인

## 8. 2단계: 저장한 프롬프트로 새 원고 읽기

같은 탭의 아래쪽 영역에서 새 원고를 넣어 다시 생성할 수 있습니다.

### 8-1. `저장된 voice prompt`

보통 바로 위 단계에서 저장에 성공했다면 파일이 이미 연결됩니다.

만약 나중에 다시 실행한 경우:
- 이 칸에서 기존 `.pt` 파일을 선택하면 됩니다.

중요:
- 반드시 이 프로그램에서 만든 `.pt` 파일을 사용하는 것이 안전합니다.

### 8-2. `재사용할 원고`

이 칸에는 새로 읽힐 문장을 넣습니다.

예:

```text
이번 영상에서는 제품 설치가 끝난 뒤 처음으로 확인해야 할 항목을 순서대로 설명하겠습니다.
```

### 8-3. `저장 프롬프트로 생성`

버튼을 누르면 프로그램이:

1. `.pt` 파일을 불러옵니다.
2. 그 안의 음성 프롬프트 정보를 복원합니다.
3. 새 원고를 같은 화자 느낌으로 생성합니다.
4. 결과를 `.wav` 파일로 저장합니다.

정상 완료 시:
- `재사용 결과`에 성공 메시지 표시
- `생성된 원본 음성`에서 바로 재생 가능

## 9. 재사용으로 생성된 파일 저장 위치

저장 위치:

- [outputs](/f:/abc/qwen-tts/outputs)

파일 이름 형태:

```text
tts_prompt_xxxxxxxx.wav
```

실제로는 입력한 문장 앞부분이 파일명에 일부 반영될 수 있습니다.

## 10. 후처리 기능 사용법

이 탭에도 `원본 생성 후 후처리` 영역이 있습니다.

의미는 첫 번째 탭과 같습니다.

할 수 있는 일:
- 속도 조절
- 높낮이 조절
- 끝음 정리
- 프리셋 저장 및 재사용

사용 순서:

1. 먼저 `저장 프롬프트로 생성`으로 원본 음성을 만듭니다.
2. 아래 후처리 옵션을 조절합니다.
3. `후처리 미리듣기`를 누릅니다.
4. 결과를 듣고 필요한 경우 다시 조절합니다.

## 11. 프리셋 저장 방법

같은 후처리 세팅을 자주 쓰는 경우 아래처럼 저장합니다.

1. `프리셋 이름` 입력
2. 속도, 높낮이, 끝음 옵션 조절
3. `현재값 저장`

나중에는:

1. `저장된 프리셋` 선택
2. `프리셋 불러오기`

로 바로 복원할 수 있습니다.

프리셋 저장 위치:

- [outputs/postprocess_presets.json](/f:/abc/qwen-tts/outputs/postprocess_presets.json)

## 12. 어떤 경우에 이 기능을 써야 하나요?

`실시간 음성 복제`와의 차이를 초보자용으로 설명하면 아래와 같습니다.

`실시간 음성 복제`가 맞는 경우:
- 한두 번만 빠르게 시험할 때
- 참조 음성을 그때그때 바꿔가며 테스트할 때

`음성 프롬프트 저장/재사용`이 맞는 경우:
- 같은 목소리를 계속 써야 할 때
- 유튜브 시리즈, 강의, 안내 음성처럼 반복 제작할 때
- 화자별 설정을 파일로 보관하고 싶을 때

## 13. 실무형 추천 운영 방법

배포 문서에는 아래 운영법도 함께 넣는 것이 좋습니다.

1. 화자마다 깨끗한 참조 음성을 하나씩 준비합니다.
2. 화자마다 `.pt` 파일을 하나씩 만듭니다.
3. 파일 이름이나 보관 폴더를 화자 기준으로 정리합니다.
4. 실제 제작 때는 `.pt` 파일만 골라 새 원고를 읽힙니다.

예시 정리 방식:
- 진행자A
- 여성나레이션_차분
- 남성안내_밝음

## 14. 자주 생기는 문제와 해결 방법

### 14-1. `먼저 TTS 모델을 로드해주세요!`

원인:
- 위쪽 공통 영역에서 모델 로드를 하지 않음

해결:
- 모델 선택 후 `모델 로드`

### 14-2. `참조 음성이 필요합니다.`

원인:
- 프롬프트 저장 단계에서 음성 입력이 비어 있음

해결:
- 파일 업로드 또는 마이크 녹음

### 14-3. `프롬프트 저장 시 참조 텍스트를 입력해주세요.`

원인:
- `x-vector only`를 끄고 참조 텍스트를 비워 둠

해결:
- 참조 텍스트를 입력하거나, 꼭 필요할 때만 `x-vector only` 사용

### 14-4. `저장된 음성 프롬프트 파일을 선택해주세요.`

원인:
- 재사용 단계에서 `.pt` 파일이 선택되지 않음

해결:
- 위에서 방금 만든 파일을 연결하거나 기존 `.pt` 파일 선택

### 14-5. `올바른 voice prompt 파일이 아닙니다.`

원인:
- 다른 프로그램의 `.pt` 파일
- 파일 손상
- 잘못된 파일 선택

해결:
- 이 프로그램에서 만든 프롬프트 파일인지 확인

## 15. 초보자에게 꼭 강조할 주의점

- `.pt` 파일은 음성 재사용용 자산입니다. 지우면 같은 설정을 다시 쓰기 어렵습니다.
- 참조 음성이 좋을수록 나중에 반복 생성 품질도 안정적입니다.
- 한 번 잘 만든 프롬프트는 여러 콘텐츠 제작에 큰 시간을 절약해 줍니다.
- 본인 음성이나 사용 권한이 있는 음성만 사용해야 합니다.

## 16. 한 줄 요약

이 탭은 `한 번 만든 목소리를 저장해서 여러 번 다시 쓰는 기능`입니다. 반복 제작이 목적이라면 초보자에게 가장 먼저 익히게 해야 하는 핵심 기능입니다.
