# 음성 대 음성 변조 (RVC 실험) 사용 설명서

이 문서는 `tts_app.py`의 세 번째 탭인 `음성 대 음성 변조 (RVC 실험)` 기능을 설명합니다.

이 기능은 앞의 두 기능과 다릅니다.

앞의 두 기능:
- 텍스트를 새 음성으로 읽게 만드는 기능

이 기능:
- 이미 있는 음성 파일의 말한 내용은 유지하면서
- 다른 화자 모델의 목소리 쪽으로 변환하는 기능

쉽게 말하면:
- `원고를 읽게 만드는 기능`이 아니라
- `기존에 녹음된 음성을 다른 목소리 모델로 바꾸는 기능`입니다

## 1. 먼저 꼭 읽어야 할 주의 사항

이 기능은 코드에서도 `실험`으로 표시되어 있습니다.

의미:
- 환경에 따라 설치가 잘 안 될 수 있습니다.
- 첫 번째, 두 번째 기능보다 준비물이 많습니다.
- 처음 사용자에게는 가장 마지막에 안내하는 것이 좋습니다.

특히 Windows에서는 아래 조합이 권장됩니다.

- Python `3.10`
- 별도 가상환경
- TTS 기능과 분리된 설치 환경

## 2. 이 기능으로 할 수 있는 일

예를 들어:

1. 내가 읽은 원본 음성 파일이 있고
2. 특정 RVC 모델이 있으며
3. 그 모델의 화자 느낌으로 바꾸고 싶다면

이 탭에서 입력 음성을 변환할 수 있습니다.

결과:
- 말한 내용은 비슷하게 유지
- 목소리 성향은 RVC 모델 쪽으로 변환

## 3. 준비해야 할 것

이 기능은 일반 TTS보다 준비물이 더 많습니다.

### 3-1. 필수 준비물

- 변조할 입력 음성 파일
- RVC 모델 파일 `*.pth`
- HuBERT 모델 파일 `hubert_base.pt`

### 3-2. 경우에 따라 필요한 준비물

- Feature Index 파일 `*.index`
- RMVPE 파일 `rmvpe.pt`

주의:
- `F0 추출 방식`을 `rmvpe`로 쓸 경우 `rmvpe.pt`가 필요합니다.

## 4. 설치 방법

먼저 TTS 기본 패키지를 설치합니다.

```powershell
pip install -r requirements-tts.txt
```

그 다음 RVC 관련 패키지를 설치합니다.

```powershell
pip install -r requirements-rvc.txt
pip install --no-deps git+https://github.com/RVC-Project/Retrieval-based-Voice-Conversion
```

권장 추가 사항:

```powershell
python -m pip install pip==24.0
```

배포 문서에는 아래 문장을 함께 넣는 것이 좋습니다.

- 이 기능은 되도록 Python 3.10 전용 가상환경에서 사용하세요.
- 기존 ASR 또는 일반 TTS 환경과 섞지 않는 편이 안전합니다.

## 6. 실행 방법

```powershell
python tts_app.py
```

또는 현재 저장소 예시:

```powershell
.\.venv-rvc\Scripts\python.exe tts_app.py
```

실행 후:

1. 브라우저에서 프로그램 열기
2. 세 번째 탭 `음성 대 음성 변조 (RVC 실험)`으로 이동

이 기능은 TTS 모델 로드와 직접 연결되지는 않지만, 같은 앱 안에서 함께 제공됩니다.

## 7. 먼저 `RVC 런타임 상태` 확인하기

이 탭에 들어가면 맨 위에 `RVC 런타임 상태`가 있습니다.

여기서 확인할 것:
- RVC 패키지 사용 가능 여부
- 모델 폴더 위치
- Index 폴더 위치
- HuBERT 준비 여부
- RMVPE 준비 여부

문제가 있을 때는 `RVC 상태 새로고침`을 눌러 다시 확인하세요.

정상 예시 의미:
- RVC 패키지가 설치됨
- HuBERT 있음
- RMVPE 있음

비정상 예시 의미:
- RVC 패키지 자체가 설치되지 않음
- HuBERT 없음
- RMVPE 없음

## 8. 화면 항목별 설명

### 8-1. `변조할 입력 음성`

이 칸에는 목소리를 바꿀 원본 음성을 넣습니다.

방법:
- 파일 업로드
- 마이크 녹음

권장:
- 말소리가 분명한 파일
- 너무 긴 파일보다는 짧은 문장 단위 테스트부터 시작

### 8-2. `RVC 모델 (.pth)`

가장 중요한 파일입니다.

의미:
- 어떤 목소리 방향으로 바꿀지 정하는 모델

없으면 변환이 시작되지 않습니다.

### 8-3. `Feature Index (.index, 선택)`

선택 사항입니다.

의미:
- 일부 모델에서 음색 안정성이나 품질에 도움될 수 있는 추가 파일

없어도 동작할 수 있습니다.

### 8-4. `HuBERT base (.pt, 선택)`

선택이라고 표시되지만, 실제로 기본 위치에도 없고 업로드도 안 하면 변환이 실패합니다.

즉 실무적으로는 필수라고 생각하는 편이 안전합니다.

### 8-5. `RMVPE (.pt, 선택)`

`F0 추출 방식`이 `rmvpe`일 때 필요합니다.

기본값이 `rmvpe`이므로, 초보자는 사실상 준비해 두는 것이 좋습니다.

### 8-6. `화자 ID`

기본값은 `0`입니다.

대부분의 단일 화자 모델은 `0` 그대로 쓰는 경우가 많습니다.

모델에 따라 다를 수 있으므로:
- 결과가 이상하면 학습된 모델의 안내값을 확인해야 합니다.

### 8-7. `키 변경 (반음)`

입력 음성의 높낮이를 반음 단위로 올리거나 내립니다.

예:
- `+2`: 조금 더 높게
- `-2`: 조금 더 낮게

초보자 권장:
- 처음에는 `0`으로 시작

### 8-8. `F0 추출 방식`

선택지:
- `rmvpe`
- `harvest`
- `pm`
- `crepe`

초보자 권장:
- 먼저 기본값 `rmvpe`

단, 이 경우 `rmvpe.pt`가 준비되어 있어야 합니다.

### 8-9. `Index 비율`

RVC 모델의 index 정보를 얼마나 반영할지 정하는 값입니다.

기본값:
- `0.75`

초보자 권장:
- 처음에는 기본값 유지

### 8-10. `Filter Radius`

출력 안정화에 관련된 옵션입니다.

초보자 권장:
- 기본값 유지

### 8-11. `출력 샘플레이트`

선택지:
- 모델 기본값
- 32000 Hz
- 40000 Hz
- 48000 Hz

초보자 권장:
- 먼저 `모델 기본값`

### 8-12. `RMS Mix Rate`

출력 음량 느낌과 관련된 혼합 옵션입니다.

초보자 권장:
- 기본값 유지

### 8-13. `Protect`

일부 발음이나 원래 음색 특성을 얼마나 보호할지 관련된 값입니다.

초보자 권장:
- 기본값 유지

## 9. 초보자용 가장 쉬운 사용 순서

1. Python 3.10 가상환경 준비
2. `requirements-tts.txt`, `requirements-rvc.txt` 설치
3. RVC 패키지 추가 설치
4. `hubert_base.pt`, `rmvpe.pt`, `RVC 모델(.pth)` 준비
5. `python tts_app.py` 실행
6. `음성 대 음성 변조 (RVC 실험)` 탭 열기
7. `RVC 런타임 상태` 확인
8. `변조할 입력 음성` 업로드
9. `RVC 모델(.pth)` 업로드
10. 필요하면 `.index`, `HuBERT`, `RMVPE` 파일도 업로드
11. 나머지 옵션은 기본값 유지
12. `음성 변조 실행` 클릭
13. 결과 음성을 재생해 확인


## 11. 결과 메시지 읽는 방법

성공 시 결과창에는 보통 아래 정보가 들어갑니다.

- 사용한 모델 이름
- 사용한 Index 파일 이름 또는 미사용 여부
- 출력 샘플레이트
- 경우에 따라 처리 시간 단서

실패 시에는:
- 어떤 파일이 빠졌는지
- 모델 로드가 왜 실패했는지
- 런타임 패키지가 없는지

같은 정보가 표시됩니다.

## 12. 자주 생기는 문제와 해결 방법

### 12-1. `RVC 패키지가 설치되지 않았습니다.`

원인:
- RVC 관련 패키지 미설치
- Python 버전 또는 환경 충돌

해결:
- 별도 가상환경에서 다시 설치
- Python 3.10 사용 권장

### 12-2. `RVC 모델(.pth) 파일이 필요합니다.`

원인:
- 변환용 모델 파일이 없음

해결:
- `.pth` 파일 업로드 또는 준비

### 12-3. `HuBERT base 모델이 없습니다.`

원인:
- `hubert_base.pt`가 업로드되지 않았고 기본 위치에도 없음

해결:
- 업로드하거나 아래 경로에 배치
- [models/rvc/assets/hubert](/f:/abc/qwen-tts/models/rvc/assets/hubert)

### 12-4. `RMVPE 모델이 없습니다.`

원인:
- `F0 추출 방식`이 `rmvpe`인데 `rmvpe.pt`가 없음

해결:
- `rmvpe.pt` 업로드 또는 아래 경로에 배치
- [models/rvc/assets/rmvpe](/f:/abc/qwen-tts/models/rvc/assets/rmvpe)

### 12-5. 결과 품질이 이상합니다

가능한 원인:
- 입력 음성과 RVC 모델의 성별, 톤, 도메인이 잘 맞지 않음
- `키 변경` 값이 과도함
- 모델 품질 자체 문제

해결:
- `키 변경`을 `0`으로 되돌리고 다시 테스트
- 더 짧고 깨끗한 입력 음성으로 재시도
- 해당 RVC 모델에 맞는 권장 옵션 확인

## 13. 초보자에게 안내할 현실적인 기대치

이 기능은 마법처럼 항상 완벽하게 바뀌지 않습니다.

결과에 영향을 주는 요소:
- 입력 음성 품질
- RVC 모델 품질
- 모델이 학습된 화자 스타일
- 옵션값 조합

따라서 초보자에게는 아래처럼 안내하는 것이 좋습니다.

- 먼저 짧은 음성으로 시험하세요.
- 기본값으로 1회 실행해 보세요.
- 잘 되면 그다음에 `키 변경`만 조금 조절하세요.
- 품질이 크게 흔들리면 모델 자체의 적합성을 먼저 의심하세요.

## 14. 사용 목적이 헷갈릴 때 구분법

이 기능을 써야 하는 경우:
- 이미 녹음된 음성 파일을 다른 목소리 느낌으로 바꾸고 싶을 때

이 기능이 아닌 다른 탭을 써야 하는 경우:
- 텍스트를 처음부터 읽게 하고 싶을 때
- 참조 음성 기반으로 새 문장을 생성하고 싶을 때
- 목소리를 저장해서 반복 재사용하고 싶을 때

## 15. 한 줄 요약

이 탭은 `이미 있는 음성을 다른 목소리 모델로 바꾸는 실험 기능`입니다. 초보자에게는 필수 준비물과 환경 제약이 많다는 점을 먼저 분명히 안내해야 합니다.
