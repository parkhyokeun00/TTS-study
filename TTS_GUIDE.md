# Qwen3-TTS 사용 설명서

이 문서는 이 폴더에 추가된 `Qwen3-TTS` 기반 텍스트 음성 변환 및 음성 복제 기능의 사용법을 설명합니다.

핵심 대상은 다음입니다.
- 한국어 원고를 음성으로 만들고 싶은 사용자
- 참조 음성을 기반으로 같은 화자 느낌을 재사용하고 싶은 사용자
- 한 번 만든 음성 설정을 여러 콘텐츠에 반복 적용하고 싶은 사용자

## 1. 프로그램 개요

이 프로젝트에는 현재 두 계열 기능이 있습니다.

- ASR: `음성 -> 텍스트`
- TTS: `텍스트 -> 음성`

TTS 관련 실행 파일은 다음입니다.

- [tts_app.py](/f:/abc/qwen-tts/tts_app.py): Gradio 웹 UI
- [tts.py](/f:/abc/qwen-tts/tts.py): Qwen3-TTS 로딩 및 생성 로직
- [requirements-tts.txt](/f:/abc/qwen-tts/requirements-tts.txt): TTS 전용 의존성

## 2. 실행 방법

## 3. 화면 구성

`tts_app.py`를 실행하면 다음 순서로 사용합니다.

### 상단 공통 영역

- `모델 선택`
- `모델 로드`
- `상태`
- `출력 언어`

기본 권장 모델:
- `Qwen3-TTS-1.7B Base (음성 복제)`

이 모델은 다음 용도에 가장 적합합니다.
- 참조 음성 기반 voice clone
- 저장한 voice prompt 재사용

### 탭 1: 실시간 음성 복제

입력 요소:
- `합성할 원고`
- `참조 음성`
- `참조 음성 텍스트`
- `x-vector only 모드 사용`

출력 요소:
- `생성 결과`
- `생성된 음성`

### 탭 2: 음성 프롬프트 저장/재사용

입력 요소:
- `프롬프트용 참조 음성`
- `프롬프트용 참조 텍스트`
- `x-vector only 모드 사용`
- `저장된 voice prompt`
- `재사용할 원고`

출력 요소:
- `프롬프트 상태`
- `재사용 결과`
- `생성된 음성`

## 4. 가장 기본적인 사용 순서

### A. 바로 음성 복제해서 한 문장 생성

1. `python tts_app.py` 실행
2. 브라우저에서 TTS 페이지 열기
3. `Qwen3-TTS-1.7B Base (음성 복제)` 선택
4. `모델 로드` 클릭
5. `출력 언어`를 `한국어`로 설정
6. `합성할 원고` 입력
7. `참조 음성` 업로드 또는 마이크 녹음
8. 가능하면 `참조 음성 텍스트`도 입력
9. `음성 생성` 클릭

결과:
- 생성된 음성이 재생 가능 상태로 표시됨
- 파일은 `outputs/` 폴더에 `.wav`로 저장됨

### B. 같은 목소리를 여러 문장에 반복 사용

1. `음성 프롬프트 저장/재사용` 탭으로 이동
2. `프롬프트용 참조 음성` 업로드
3. `프롬프트용 참조 텍스트` 입력
4. `음성 프롬프트 저장` 클릭
5. 생성된 `.pt` 파일 확보
6. `재사용할 원고`에 새 문장 입력
7. 같은 탭에서 `저장 프롬프트로 생성` 클릭

결과:
- 저장된 음성 특징을 재사용해 새 문장을 음성으로 생성
- 참조 음성을 매번 다시 업로드하지 않아도 됨

## 5. 음성을 저장하고 재사용하는 파이프라인

이 프로젝트에서 가장 중요한 흐름입니다.

### 5-1. 목적

한 번 참조 음성에서 화자 특성을 추출해두고, 이후에는 그 결과를 다시 불러와 여러 원고를 같은 목소리 계열로 읽게 하는 것입니다.

### 5-2. 사용자 관점 파이프라인

1. 참조 음성 준비
2. 필요하면 참조 음성 텍스트 준비
3. `음성 프롬프트 저장` 실행
4. `.pt` 파일 생성
5. 이후 다른 날에도 `.pt` 파일 불러오기
6. 새 원고 입력
7. `저장 프롬프트로 생성` 실행

즉, `원본 참조 음성`을 매번 다시 분석하지 않고, 분석 결과를 저장한 뒤 재사용하는 구조입니다.

### 5-3. 코드 기준 파이프라인

관련 함수는 [tts.py](/f:/abc/qwen-tts/tts.py)에 있습니다.

#### 1) 참조 음성 정규화

함수:
- [tts.py](/f:/abc/qwen-tts/tts.py)

내부 함수:
- `_normalize_audio_input(...)`

역할:
- 업로드 또는 마이크 입력을 numpy 오디오로 받음
- 스테레오면 모노로 변환
- 값 범위를 `float32` 기준으로 정리
- `(audio_array, sample_rate)` 형태로 표준화

#### 2) 음성 프롬프트 생성

함수:
- `save_voice_prompt(...)`

역할:
- `Qwen3TTSModel.create_voice_clone_prompt(...)` 호출
- 참조 음성에서 화자 특징을 추출
- 필요 시 참조 텍스트를 함께 사용
- 결과를 `VoiceClonePromptItem` 목록으로 만듦

정확히 말하면 여기서 저장되는 것은 “참조 음성 파일 자체”가 아니라, 모델이 재사용할 수 있는 “클론용 프롬프트 정보”입니다.

#### 3) 프롬프트 파일 저장

함수:
- `save_voice_prompt(...)`

저장 형식:
- `.pt`

저장 위치:
- `outputs/`

저장 내용:
- `{"items": [asdict(item) for item in items]}`

즉, `VoiceClonePromptItem`들을 딕셔너리로 바꿔서 Torch 직렬화 파일로 저장합니다.

#### 4) 저장된 프롬프트 재로드

함수:
- `generate_from_prompt_file(...)`

역할:
- 사용자가 선택한 `.pt` 파일을 `torch.load(...)`로 읽음
- 내부 `items` 목록을 다시 `VoiceClonePromptItem(...)` 객체로 복원

#### 5) 저장 프롬프트로 새 음성 생성

함수:
- `generate_from_prompt_file(...)`

역할:
- `self.model.generate_voice_clone(...)` 호출
- 이때 `ref_audio` 대신 `voice_clone_prompt=items` 사용
- 새 텍스트를 같은 화자 특성으로 읽음

#### 6) 최종 음성 저장

함수:
- `_save_wav(...)`

역할:
- 생성 결과를 `outputs/` 폴더에 `.wav`로 저장
- UI에는 해당 파일 경로를 반환

## 6. `x-vector only` 모드와 일반 모드 차이

### 일반 모드

입력:
- 참조 음성
- 참조 텍스트

특징:
- 더 정밀한 복제
- 원래 발화 스타일과 발음 습관까지 반영될 가능성이 높음
- 이 프로젝트에서는 기본 권장 방식

### x-vector only 모드

입력:
- 참조 음성만 필요

특징:
- 참조 텍스트 없이 빠르게 화자 특성만 반영
- 편하지만 품질과 일관성은 일반 모드보다 떨어질 수 있음
- 참조 텍스트를 준비하기 어려울 때 사용

코드상으로는 `x_vector_only_mode=True`일 때 이 경로를 사용합니다.

## 8. 품질 좋게 쓰는 방법

권장:
- 참조 음성은 잡음이 적은 파일 사용
- 3초보다 너무 짧지 않게 준비
- 한국어 문장 발음이 또렷한 샘플 사용
- 가능하면 참조 음성과 참조 텍스트를 정확히 맞춤
- 콘텐츠 본문은 너무 길면 문단 단위로 나눠 생성

좋지 않은 예:
- 배경음악이 큰 샘플
- 여러 사람이 같이 말하는 샘플
- 참조 텍스트와 실제 음성이 다른 경우
- 지나치게 긴 문장을 한 번에 합성하는 경우

## 9. 자주 생기는 문제

### 모델 로드가 느림

원인:
- 첫 다운로드
- CPU 로드
- 대형 모델 초기화

확인:
- 상태창에 `GPU (CUDA)`인지 확인

### GPU를 못 잡음

확인:
- CUDA 지원 PyTorch 설치 여부
- `python -c "import torch; print(torch.cuda.is_available())"`

### `SoX` 경고 발생

설명:
- Windows에서 `SoX` 실행파일이 PATH에 없으면 경고가 뜰 수 있음

조치:
- 필요 시 SoX 설치 후 PATH 등록

### 프롬프트 파일을 불러왔는데 실패

확인:
- `.pt` 파일이 이 프로그램에서 생성된 파일인지
- 파일이 손상되지 않았는지
- 다른 모델 계열에서 만든 프롬프트를 섞지 않았는지

## 10. 콘텐츠 제작용 추천 운영 방식

실무적으로는 다음 흐름을 추천합니다.

1. 화자별 참조 음성 샘플 확보
2. 각 화자마다 `voice_prompt_*.pt` 생성
3. 프롬프트 파일 이름을 의미 있게 정리
4. 콘텐츠 원고를 짧은 문단 단위로 분리
5. 프롬프트 파일별로 반복 생성
6. 생성된 `.wav`를 편집 툴에서 후처리

예:
- `voice_prompt_host_main.pt`
- `voice_prompt_narration_soft.pt`
- `voice_prompt_short_ad.pt`

이 방식이면 같은 목소리 톤을 유지하면서 여러 영상에 재사용하기 좋습니다.
